# 通用表格智能校验与变更说明生成器 使用说明书

## 1. 这是什么工具

这是一个本地运行的表格分析工具，用来比较两份 Excel 或 CSV 文件的差异，并自动整理出一份容易阅读的变化说明。

它主要能帮你做四件事：

- 检查表格中的缺失值
- 检查主键是否重复
- 对比旧版和新版表格的新增、删除、修改内容
- 生成一段中文变更说明，并导出报告

这个工具适合第一次上手的普通用户使用，不需要数据库，也不需要注册登录。

## 2. 适用场景

这个工具适合下面这些场景：

- 比较两版商品表、人员名单、活动报名表、配置表
- 检查别人发来的新版表格改了哪些内容
- 在提交、交接、汇报前，快速整理一份变更说明
- 在演示或面试时，展示表格校验和版本对比能力

如果你的文件是结构化表格，且旧版和新版之间有可对应的主键列，这个工具就很适合。

## 3. 支持的文件类型

- `.csv`
- `.xlsx`
- `.xls`

建议：

- 第一行要是表头
- 新旧两份文件尽量包含相同的主键列
- 一份旧表配一份新表，最适合做版本对比

## 4. 安装方式

### 4.1 环境要求

- Python 3.10 及以上

### 4.2 安装依赖

在项目根目录执行：

```bash
pip install -r requirements.txt
```

如果你想先创建虚拟环境，也可以这样做：

```bash
python -m venv .venv
```

Windows PowerShell 激活虚拟环境：

```powershell
.venv\Scripts\Activate.ps1
```

然后再安装依赖：

```bash
pip install -r requirements.txt
```

## 5. 运行方式

在项目根目录执行：

```bash
streamlit run app.py
```

启动后，终端会显示本地访问地址。通常可以在浏览器打开：

```text
http://localhost:8501
```

## 6. 页面能做什么

当前版本的主页面支持两种数据来源：

- 上传你自己的旧表和新表
- 直接切换并加载内置示例数据

分析完成后，页面会显示这些内容：

- 汇总指标卡片
- 校验概览
- 差异分析
- AI 变更摘要
- 详细数据
- HTML 报告下载

说明：

- AI 摘要是按需生成的，不会在一开始自动生成
- 如果没有配置大模型接口，系统会自动使用本地模板生成中文摘要

## 7. 基本操作流程

### 方式一：使用你自己的文件

1. 打开页面左侧边栏。
2. 保持“使用内置示例”为关闭状态。
3. 上传旧版本文件。
4. 上传新版本文件。
5. 点击“加载文件”。
6. 在“主键列”下拉框中选择用于对比的主键。
7. 点击“开始分析”。
8. 在主页面查看汇总结果、差异结果和详细数据。
9. 如果需要更自然的说明文字，点击“生成 AI 摘要”。
10. 如果需要对外发送结果，点击下载 HTML 报告。

### 方式二：使用内置示例

1. 打开页面左侧边栏。
2. 打开“使用内置示例”。
3. 选择示例场景。
4. 系统会自动加载对应的旧表和新表。
5. 确认推荐主键。
6. 点击“开始分析”。
7. 查看摘要、差异和详情。

当前内置示例包括：

- 商品表
- 人员名单
- 活动报名表

这些示例适合快速演示功能，不用自己准备文件。

## 8. 结果怎么看

### 8.1 分析摘要

页面顶部会先显示几张摘要卡片，帮助你快速看重点，包括：

- 空值问题数
- 重复主键数
- 新增行数
- 删除行数
- 修改行数

如果你只想先判断“有没有明显问题”，看这一块就够了。

### 8.2 校验概览

这一部分会分别展示旧表和新表的基础信息与校验结果，例如：

- 行数
- 列数
- 缺失值情况
- 重复主键情况
- 异常数值列数量
- 疑似类型问题数量

### 8.3 差异分析

这一部分会告诉你：

- 新表比旧表新增了哪些行
- 删除了哪些行
- 哪些行被修改了
- 哪些列发生了变化
- 是否新增列或删除列

### 8.4 AI 摘要

这部分会把前面的结构化结果整理成一段更适合给同事、领导或面试官看的中文说明。

如果你没有配置 AI 接口，也不用担心，系统会自动给出一段本地模板摘要。

### 8.5 明细数据

这一部分默认折叠，避免首页太长。打开后可以查看：

- 缺失值明细
- 重复主键明细
- 新增行与删除行
- 字段变更明细

## 10. 常见问题

### 10.1 为什么上传后提示文件读取失败

常见原因：

- 文件为空
- 文件扩展名是对的，但内容已经损坏
- CSV 编码无法识别
- Excel 文件格式不完整

建议处理方式：

- 确认文件能在 Excel 或表格软件里正常打开
- 重新另存为 `.xlsx` 或 UTF-8 编码的 `.csv`
- 确认第一行是表头

### 10.2 为什么不能开始分析

常见原因：

- 还没有成功加载旧表和新表
- 还没有选择主键列
- 两个文件没有共同列

建议先看左侧边栏是否已经显示当前文件名，再确认主键列是否已选中。

### 10.3 为什么分析时报主键相关错误

这个工具依赖主键来判断“哪一行对应哪一行”。如果主键列有问题，就无法稳定比较。

常见原因：

- 选择了错误的主键列
- 主键列在某一份表里不存在
- 主键列中有重复值

建议：

- 优先选择编号类字段，如商品编号、员工编号、报名编号
- 不建议把姓名、邮箱、电话这类可能重复或变动的列当作主键

### 10.4 为什么 AI 摘要没有调用真实模型

当前程序只有在环境变量配置完整时，才会调用外部大模型接口。否则会自动使用本地模板摘要。

这不是报错，是正常的降级处理。

如果你只是本地演示功能，不配置也可以正常使用。

### 10.5 能不能导出 Markdown

当前页面直接提供的是 HTML 报告下载。

代码层已经支持生成 Markdown 报告，但默认页面按钮目前只放出了 HTML 下载入口。所以如果你在页面里没看到 Markdown 下载按钮，这是当前实现状态，不是你操作错了。

### 10.6 支持只上传一份文件吗

当前主流程是“旧表 + 新表”的对比分析，因此最推荐同时提供两份文件。

如果只想体验功能，建议直接使用内置示例数据。

## 11. 使用建议

- 先用内置示例熟悉页面，再上传自己的文件
- 主键尽量选唯一且稳定的编号字段
- 如果是重要业务数据，建议先看摘要，再展开明细确认
- 导出报告前，最好手动复核关键变更行

## 12. 一句话总结

如果你想快速知道两份表格哪里变了、有没有明显质量问题、并顺手生成一段中文变更说明，这个工具就是为这个场景准备的。
