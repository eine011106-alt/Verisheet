# 过程记录

这个项目目的是为了消除人工对比数据表的重复工作。前期用 ChatGPT 生成了初版 AGENTS.md，把项目目标、范围、交付物和验收标准先定好，再去实现功能和文档。其中，文档的作用非常重要，每次 Agent 新起对话时，都需要参考一个统一的文档来进行代码实现。

开发过程中，我先确定这是一个本地可运行、能完整演示的 MVP，再逐步完善核心功能和展示重点。中间也经历了几轮验收和 bug 回归修改，包括功能取舍、页面表达、文档同步等。

整个过程里我最明确的体会是，AI 和人工的边界不能乱，尤其架构不能交给 AI 来做。AI 适合整理思路、润色文案、补充摘要，但项目该怎么拆、流程怎么定、哪些能力先做、哪些边界必须保留，也就是真正影响项目稳定性和可维护性的判断，还是必须要由人工自行把关。

目前，这个项目还有一些不足之处，例如我希望 AI 可以在项目中发挥更多的作用，辅助给出修复建议、直接补全缺失的数据，但想要进一步引入 AI ，势必会经历更复杂的设计，让项目变得庞大。目前优先级最高的还是保证分析结果的准确性和可解释性，因此这些可以放到下一个版本再去完善。
