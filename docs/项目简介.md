# 项目简介

## 1. 项目名称

通用表格智能校验与变更说明生成器

## 2. 项目概述

这是一个面向 Excel / CSV 场景的本地化表格分析工具，主要解决两类高频问题：

- 表格本身是否存在明显的数据质量问题
- 新旧两个版本之间到底发生了哪些变化，如何快速说明给他人

在日常工作中，商品表、人员名单、活动报名表、运营台账、配置类表格经常被多人反复修改。传统做法往往依赖人工逐列检查、手动比对版本、再额外整理变更说明，效率低且容易遗漏。这个项目希望把“发现问题”和“说明问题”这两步串成一个完整闭环。

## 3. 项目目标

本项目的目标，是在较低使用门槛下，提供一个可本地运行、可直接演示、可稳定输出结果的 MVP 工具，让用户能够完成：

- 上传旧表和新表
- 选择主键列
- 执行单表校验与版本差异分析
- 查看结构化结果与中文摘要
- 导出可分享的分析报告

项目并不追求复杂平台能力，而是优先保证结果准确、流程清晰、演示完整。

## 4. 当前已实现的核心能力

结合当前代码实现，项目已经具备以下能力：

- 支持上传 `.csv`、`.xlsx`、`.xls` 文件
- 支持侧边栏切换内置示例数据，并自动加载
- 支持选择主键列进行稳定的行级对比
- 支持检测缺失值、重复主键、异常数值列、疑似类型不一致
- 支持输出新增行、删除行、修改行、字段变更统计
- 支持按需生成 AI 变更摘要
- 在未配置模型接口时，支持自动回退到本地模板摘要
- 支持生成并下载 HTML 报告
- 代码层已支持 Markdown 报告生成

从产品形态看，它已经不是单纯的“表格 diff 脚本”，而是一个可以完整展示输入、分析、解释、输出链路的小型工具。

## 5. 项目价值

这个项目的价值主要体现在四个方面。

第一，它将表格检查中重复、机械、容易漏掉的问题前置自动化。相比人工逐列查看，规则化校验能更快暴露空值、重复主键、异常值和类型问题。

第二，它不只解决“哪里变了”，还解决“怎么把变化讲清楚”。在结构化分析结果基础上，系统可以生成自然语言摘要，帮助用户更快完成交接、汇报和说明。

第三，它对 AI 的使用边界比较清晰。文件读取、主键判断、缺失值统计、Diff 比较这类确定性任务由规则完成，AI 主要负责摘要生成和表达增强。这样既提高了可读性，也保留了分析结果的稳定性和可解释性。

第四，它适合作为面试或演示项目提交。项目范围明确、架构简单、功能闭环完整，能够体现产品拆解、工程实现和 AI 协作边界意识。

## 6. 项目定位

该项目定位为“通用表格校验与变更说明工具”，而不是自动修改原始数据的修复系统，也不是重型数据治理平台。

当前版本更强调：

- 本地可运行
- 普通用户可理解
- 结果稳定可解释
- 演示链路完整

这种定位有意控制了系统复杂度，使项目重点始终落在真实问题的解决效率上。

## 7. 技术思路简述

项目采用 `Python + Streamlit + pandas` 的实现方式：

- `app.py` 负责页面交互与流程编排
- `loader` 负责文件识别与加载
- `validator` 负责单表校验
- `differ` 负责版本差异分析
- `llm_summary` 负责自然语言摘要生成
- `reporter` 负责报告输出

整体设计遵循一个核心原则：确定性的分析交给规则，不确定性的表达交给 AI。

## 8. 实际应用场景

这个工具适合用于：

- 商品资料版本更新比对
- 员工名单或客户名单核对
- 活动报名表前后版本审查
- 配置表、映射表、小型运营数据表复核
- 需要快速给同事、领导或面试官展示“改了什么”的场景

对于需要快速识别问题、快速形成说明、又不想搭建复杂系统的场景，这个项目有比较直接的实用价值。

## 9. 后续规划方向

在当前 MVP 基础上，后续可以继续沿着“更准确、更省事、更智能”三个方向扩展：

- 增加更多字段级校验规则，例如日期格式、枚举值范围、必填列配置
- 增加风险分级和重点问题提示，让结果更适合业务人员快速判断
- 在页面中补充 Markdown 报告下载入口
- 支持保存历史分析结果，方便复核和回看
- 增加更细粒度的字段变更展示和筛选能力
- 优化 AI 提示词，让摘要更贴近业务沟通场景
- 加入 AI 自动补全功能，例如根据历史值或上下文给出缺失字段补全建议
- 加入 AI 辅助修复建议，例如提示某列疑似类型异常、推荐标准化处理方式
- 支持用户自定义校验规则和模板化报告输出

其中，“AI 自动补全”和“AI 辅助修复建议”会是比较有价值的下一步方向，但这类能力仍然应建立在规则分析结果已经明确的前提下，而不是直接替代核心判断。

## 10. 总结

通用表格智能校验与变更说明生成器，本质上是在解决一个很常见但长期低效的问题：表格改动很多，但人工检查和说明都很耗时。

这个项目用较轻的技术方案，把文件加载、规则校验、版本对比、AI 摘要和报告导出串成了一条完整链路。它既有明确的现实价值，也保留了清晰的扩展空间，是一个适合继续打磨、也适合直接展示的实用型项目。
